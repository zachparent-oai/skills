# Testing conventions (`zach-stack`)

The testing guidance for `zach-stack` was split into focused files under `references/testing/` to keep recommendations more relevant by project type and reduce context load.

## Use these files

- `references/testing/testing.md` for shared testing principles and a routing map.
- `references/testing/cli.md` for CLI testing conventions (Typer/Click, command style, examples).
- `references/testing/web-ui-e2e.md` for Playwright CLI exploration and browser E2E guidance (JS/TS + Streamlit/Python).
